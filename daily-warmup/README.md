# Daily JS Warmup (10–15 min before LeetCode)

Goal: build muscle memory for JS syntax so during the exam you don't fight the language.

## Routine (do every day in order)

1. **warmup-01-functions.js** — function declarations, arrow functions, params, return
2. **warmup-02-arrays.js** — create, push, map, filter, reduce, loops, indexing
3. **warmup-03-strings.js** — split, join, slice, includes, char access
4. **warmup-04-objects.js** — create, access, keys/values/entries, destructuring
5. **warmup-05-class.js** — class, constructor, methods, instantiation
6. **warmup-06-mini.js** — small combined challenge (changes daily — rewrite from scratch!)
7. **warmup-07-set-map.js** — Set & Map (dedupe, lookup, frequency)
8. **warmup-08-sort-math.js** — sort comparator, Math, spread/rest
9. **warmup-09-loops-recursion.js** — while, two-pointer, recursion, ternary

Each file has **PRACTICE QUESTIONS** at the bottom — read the examples once, then try those.

## How to run
```bash
node daily-warmup/warmup-01-functions.js
```

## Rules
- Type it out, don't copy-paste.
- Run each file and check the output.
- Day after day, **delete the file and rewrite it from memory**. That's where the gain is.
- Then move on to your LeetCode problem of the day.
